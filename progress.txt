## Multi-Platform Support (iPad + macOS)

Design spec: docs/superpowers/specs/2026-04-07-multi-platform-design.md

### Key Facts
- 27 of 30 Swift files are already cross-platform compatible
- Only 4 files need changes: project.pbxproj, BackgroundRefreshManager.swift, YouTubePlayerView.swift, ContentView.swift, JTFNewsApp.swift
- iPad is already in device family (1,2) — just needs verification
- macOS 14.0 minimum for SwiftData support
- Use #if os(iOS) / #if os(macOS) conditionals — no separate targets

### Codebase Patterns
- Xcode project generated manually (no Xcode IDE) — project.pbxproj uses AA-prefixed IDs
- SwiftData models use default initializers with property defaults (Swift 6 strict concurrency)
- New Swift files MUST be added to project.pbxproj in 4 places: PBXBuildFile, PBXFileReference, PBXGroup children, PBXSourcesBuildPhase
- IMPORTANT: Story model uses `storyHash` (not `hash`) — `hash` shadows NSObject.hash and crashes SwiftData
- IMPORTANT: StoriesResponse.source is String? (optional) — the JSON field is missing from the API
- IMPORTANT: Always do clean Release builds (iOS caches aggressively)
- Build command pattern: xcodebuild clean -scheme JTFNews ... && xcodebuild -scheme JTFNews -configuration Release ...
