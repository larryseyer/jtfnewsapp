## Codebase Patterns

- Xcode project generated manually (no Xcode IDE) — project.pbxproj uses AA-prefixed IDs
- SwiftData models use default initializers with property defaults (Swift 6 strict concurrency requirement)
- Folder structure: App/, Models/, Services/, Views/{Stories,Digest,Archive,Settings}/, Utilities/
- Shared xcscheme created so `xcodebuild -scheme JTFNews` works headless

## Progress

### APP-001: Create Xcode project scaffold with SwiftUI and SwiftData ✅
- Created JTFNews.xcodeproj with project.pbxproj and shared scheme
- iOS 17.0 deployment target, Swift 6 with strict concurrency
- SwiftData ModelContainer configured with Story, Source, Correction, Channel, ArchivedDay
- Three-tab TabView (Stories, Digest, Archive) with placeholder views
- Dark mode by default via .preferredColorScheme(.dark)
- Asset catalog with AppIcon and AccentColor placeholders
- Files created: App/JTFNewsApp.swift, App/ContentView.swift, Views/Stories/StoriesView.swift, Views/Digest/DigestView.swift, Views/Archive/ArchiveView.swift, Models/Story.swift, Models/Source.swift, Models/Correction.swift, Models/Channel.swift, Models/ArchivedDay.swift

### APP-002: Define SwiftData models for Story, Source, Correction, Channel ✅
- All models already created with full schemas in APP-001
- Story: id, hash, fact, sourceDisplay, sourceURLs (dict), audioURL, publishedAt, status, isCorrection
- Source: id, name, accuracy, bias, speed, consensus, controlType, owner, ownerDisplay
- Correction: storyId, originalFact, correctedFact, reason, correctingSources ([String]), correctedAt, type
- Channel: id, name, baseURL (defaults to https://jtfnews.org)
- ArchivedDay: date, rawText, isIndexed
- All registered in ModelContainer in JTFNewsApp.swift

### APP-003: Build DataService to fetch and parse stories.json and corrections.json ✅
- Created DataService as an actor for thread safety (Swift 6 strict concurrency)
- Lightweight Codable DTOs (StoryDTO, CorrectionDTO, StoriesResponse) for JSON parsing, mapped into SwiftData models
- Upsert logic: matches stories by hash, corrections by storyId
- 5-minute cooldown via UserDefaults timestamps
- ISO8601 date parsing with fractional seconds fallback
- All methods parameterized by Channel.baseURL
- Pattern: new files must be added to project.pbxproj in 4 places (PBXBuildFile, PBXFileReference, PBXGroup children, PBXSourcesBuildPhase)

### APP-004: Build FeedService to parse feed.xml for source metadata ✅
- FeedService actor with fetchSources() method parameterized by baseURL
- FeedXMLParser uses Foundation XMLParser with namespace-aware parsing
- Handles jtf:source elements, deduplicates by source name
- Creates/updates Source SwiftData models
- SourceDTO as lightweight transfer object
- Files: Services/FeedService.swift

### APP-005: Build Stories tab with story cards showing source transparency ✅
- StoriesView uses @Query sorted by publishedAt descending
- StoryCard displays fact, source badges (color-coded by rating), ownership line, relative time
- Corrections shown with red border, strikethrough original, arrow to corrected
- Pull-to-refresh via .refreshable + force param to bypass cooldown
- FlowLayout custom Layout for wrapping source badges
- parseSourceDisplay() parses "BBC 9.4|9.2 · NPR 9.2|9.0" format
- Loading/empty states handled
- Dark mode with muted colors and generous whitespace
- Files: Views/Stories/StoriesView.swift (updated), Views/Stories/StoryCard.swift (new)
