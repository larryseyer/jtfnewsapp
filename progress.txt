## Codebase Patterns

- Xcode project generated manually (no Xcode IDE) — project.pbxproj uses AA-prefixed IDs
- SwiftData models use default initializers with property defaults (Swift 6 strict concurrency requirement)
- Folder structure: App/, Models/, Services/, Views/{Stories,Digest,Archive,Settings}/, Utilities/
- Shared xcscheme created so `xcodebuild -scheme JTFNews` works headless
- New Swift files MUST be added to project.pbxproj in 4 places: PBXBuildFile, PBXFileReference, PBXGroup children, PBXSourcesBuildPhase

## Bugs Found During First App Testing (2026-04-06)

These bugs were identified by running the app in iOS Simulator for the first time:

1. Stories tab shows empty on first launch, pull-to-refresh doesn't work on empty state
2. Whitepaper link goes to jtfnews.org homepage instead of whitepaper.html
3. Privacy Policy link goes to jtfnews.org homepage; https://jtfnews.org/privacy returns 404
4. YouTube video player shows Error 153 (video player configuration error)
5. "Watch video on YouTube" link in YouTube error overlay does nothing (no WKNavigationDelegate)
6. Archive day view shows raw pipe-delimited text instead of formatted stories
7. Pull-to-refresh visual feedback missing (no scroll-down animation on empty state)
