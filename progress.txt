## Seed context (read before first iteration)

Today is **2026-04-14**. The iPad Release build shows the Digest tab stuck at **April 8** while Stories/Archive correctly show Apr 13/14. Investigation produced three compounding root causes:

1. **Production `podcast.xml` has unresolved git merge conflict markers** at lines 1110, 1204, 1205 (`<<<<<<< Updated upstream` / `=======` / `>>>>>>> Stashed changes`). Apple's `XMLParser` aborts at line 1110 col 2 with "StartTag: invalid element name." Everything after line 1110 is lost to the app. Apr 12 is also missing from the feed; Apr 13 has duplicate episode number 1 (content issue, flagged separately).

2. **Swift `PodcastXMLParser` silently returns whatever it managed to parse** before any `didFailWithError`. No error delegate, no throw, no banner. Silent partial data is not acceptable for a Truth-seeking app.

3. **`DigestView` fetches once per process** (`.task` guarded by `hasLoadedOnce`) with no scenePhase / onAppear / reconnect hook, and no cross-check against the canonical `monitor.json.last_date`. A partial first fetch is permanent until pull-to-refresh.

There are also two smaller Truth hazards in the same parser worth fixing while we are in there:

4. Items with empty `<enclosure url>` are silently dropped at PodcastService.swift:177.
5. Items with unparseable `<pubDate>` fall back to `Date()` at line 173, which — because the list sorts newest first — would float a malformed item to the top.
